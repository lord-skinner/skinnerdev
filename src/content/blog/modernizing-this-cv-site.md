This site started as a static HTML, SCSS, and jQuery project that served me well for years.

As my work shifted more toward platform engineering, cloud architecture, and modern developer tooling, it made sense for my own site to reflect that standard.

## Why Modernize

I wanted three things:

1. A maintainable codebase that is easier to evolve.
2. A cleaner publishing workflow for long-form technical writing.
3. Better performance and accessibility out of the box.

## What Changed

The stack now uses:

- Astro for static-first pages
- TypeScript for typed data/content structures
- Tailwind CSS for modern styling and responsive consistency
- Content Collections for blog post management

## What Comes Next

The immediate roadmap is:

- Add deeper post templates for architecture writeups
- Add practical engineering notes and tutorials from current work
- Continue tuning design parity while keeping the modern stack

If you are modernizing a legacy personal site, start with structure and content modeling first, then layer styling and interactivity afterward.
